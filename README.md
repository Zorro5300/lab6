# Лабораторная работа 6: Создание внутренней формы представления программы

**Вариант:** 8 (арифметические выражения)

**Цель работы**
Изучить методы построения внутреннего представления программы (ВПП) на основе контекстно-свободной грамматики, реализовать синтаксический анализатор методом рекурсивного спуска и преобразовать арифметические выражения в тетрады и ПОЛИЗ.

**Вариант задания**
**Язык программирования:** Упрощённый язык арифметических выражений.

**Полное определение КС-грамматики:**
```
E → TA
A → ε | + TA | - TA
T → FB
B → ε | * FB | / FB
F → num | id | (E)
id → letter {letter | digit}
num → digit {digit}
```

**Примеры верных строк:**
- `2 + 3 * 4`
- `(2 + 3) * 4`
- `a + b`
- `10 % 3 + 5`
- `x * (y - 1)`

**Лексические и синтаксические ошибки**
**Диаграмма лексера**
Лексический анализатор выделяет следующие классы токенов:
- **INTEGER** – последовательность цифр
- **IDENTIFIER** – буква, за которой следуют буквы, цифры или подчёркивания
- **OPERATOR_PLUS** – `+`
- **OPERATOR_MINUS** – `-`
- **OPERATOR_MUL** – `*`
- **OPERATOR_DIV** – `/`
- **OPERATOR_MOD** – `%`
- **LPAREN** – `(`
- **RPAREN** – `)`
- **WHITESPACE** – пробелы, табы, переводы строк
- **ERROR** – недопустимый символ

**ПОЛИЗ (польская инверсная запись)**
ПОЛИЗ формируется с использованием стека операций (алгоритм Дейкстры). Для выражений, состоящих исключительно из целых чисел, также вычисляется значение.

**Инструкция по запуску**
*Требования*
- Python 3.8 или выше
- Установленные зависимости: `PySide6`

**Установка зависимостей**
```bash
pip install -r requirements.txt
```
**Запуск программы**
```bash
python main.py
```
 **Использование**
1. Откроется главное окно редактора.
2. Введите арифметическое выражение в текстовую область.
3. Нажмите кнопку **«Анализ арифметики»** (или клавишу **F8**) для запуска анализа.
4. Результаты отобразятся на вкладках:
   - **«Ошибки»** – список лексических и синтаксических ошибок.
   - **«Тетрады»** – таблица сгенерированных тетрад.
   - **«ПОЛИЗ»** – текстовая запись ПОЛИЗ и вычисленное значение (если выражение состоит только из целых чисел).
5. Для обычного анализа Rust-кода используйте кнопку **«Пуск»** (F5).

**Реализованные возможности**
1. Лексический анализ арифметических выражений с обнаружением недопустимых символов.
2. Синтаксический анализ методом рекурсивного спуска с обработкой ошибок (пропущенный операнд, лишние скобки, неверный символ).
3. Генерация тетрад (внутреннее представление) для корректных выражений.
4. Построение ПОЛИЗ и вычисление значения для выражений из целых чисел.
5. Интеграция в графический интерфейс: новые вкладки, кнопка «Анализ арифметики», отображение результатов.
6. Поддержка операций `+`, `-`, `*`, `/`, `%` с правильным приоритетом.
